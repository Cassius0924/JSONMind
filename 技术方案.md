## 1. 系统架构总览

### 1.1 架构设计原则

- **分层架构**：清晰的数据层、逻辑层、渲染层分离
- **单向数据流**：确保数据一致性和可追溯性
- **组件化设计**：高内聚低耦合的模块划分
- **性能优先**：虚拟化渲染、懒加载、增量更新

## 2. 技术栈选型

### 2.1 前端框架
**选型：React 18.x + TypeScript 5.x**

**理由：**
- React 生态成熟，组件化开发效率高
- TypeScript 提供强类型支持，满足数据一致性要求（PRD 4.3）
- Hooks 机制便于状态管理和逻辑复用
- 虚拟 DOM 优化渲染性能

### 2.2 状态管理
**选型：Zustand / Jotai（轻量级）或 Redux Toolkit（复杂场景）**

**理由：**
- 需要维护复杂的数据模型树状态
- 支持时间旅行（撤销/重做功能扩展）
- 便于实现双向同步（JSON ↔ 思维导图）

**推荐：Zustand**
- 更轻量，API 简单
- 对 TypeScript 友好
- 性能优秀，无需 Context Provider

### 2.3 代码编辑器
**选型：Monaco Editor**

**理由：**
- PRD 明确要求（F-IN-1）
- VS Code 同款，功能强大
- 内置 JSON 语法高亮和校验
- 支持格式化（Pretty Print）

### 2.4 思维导图渲染引擎
**选型对比：**

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **AntV X6** | 功能强大，节点自定义能力强，支持复杂交互 | 学习曲线稍陡 | ⭐⭐⭐⭐⭐ |
| **React Flow** | React 原生，API 简洁，社区活跃 | 复杂布局算法需自行实现 | ⭐⭐⭐⭐ |
| **D3.js** | 最灵活，完全自定义 | 开发成本极高 | ⭐⭐⭐ |

**最终选型：AntV X6**

**理由：**
- 支持"组合式分裂节点"的自定义渲染（R-NODE-3）
- 内置强大的布局算法（Dagre Layout），满足水平布局和对称分布要求（R-LAYOUT-1/2）
- 提供丰富的交互 API（拖拽、缩放、平移）
- 性能优秀，支持虚拟渲染

### 2.5 UI 组件库
**选型：Ant Design 5.x**

**理由：**
- 与 AntV 系列配套，风格统一
- 提供完整的表单、按钮、对话框组件
- 支持主题定制

### 2.6 构建工具
**选型：Vite 5.x**

**理由：**
- 开发服务器启动速度极快
- 原生支持 TypeScript 和 React
- HMR（热模块替换）体验优秀

## 3. 核心模块详细设计

### 3.1 数据模型层设计

#### 3.1.1 统一节点类型定义

```typescript
// 节点类型枚举
enum NodeType {
  OBJECT = 'object',
  ARRAY = 'array',
  STRING = 'string',
  NUMBER = 'number',
  BOOLEAN = 'boolean',
  NULL = 'null'
}

// 基础节点接口
interface BaseNode {
  id: string;                    // 唯一标识（UUID）
  type: NodeType;                // 节点类型
  key: string;                   // 键名（Object 属性名或 Array 索引）
  path: string[];                // JSON 路径（如 ['data', 'users', '0']）
}

// 容器节点（Object/Array）
interface ContainerNode extends BaseNode {
  type: NodeType.OBJECT | NodeType.ARRAY;
  children: TreeNode[];          // 子节点列表
}

// 基本值节点（Primitive）
interface PrimitiveNode extends BaseNode {
  type: NodeType.STRING | NodeType.NUMBER | NodeType.BOOLEAN | NodeType.NULL;
  value: string | number | boolean | null;
}

// 联合类型
type TreeNode = ContainerNode | PrimitiveNode;

// 全局状态接口
interface AppState {
  jsonText: string;              // 原始 JSON 文本
  isValid: boolean;              // JSON 格式是否合法
  rootNode: TreeNode | null;     // 数据模型树根节点
  selectedNodeId: string | null; // 当前选中节点ID
  errorMessage: string | null;   // 错误信息
}
```

**设计要点：**
- 使用 `path` 字段追踪节点在原始 JSON 中的位置，便于编辑后重建 JSON
- 使用 `id` 字段作为唯一标识，解决 key 重复问题
- 明确区分容器节点和基本值节点，满足类型安全要求（PRD 4.3）

### 3.2 JSON 解析与转换模块

#### 3.2.1 核心转换函数

```typescript
/**
 * 将原始 JSON 转换为内部数据模型树
 * @param jsonText - JSON 字符串
 * @returns 根节点或错误信息
 */
function parseJsonToTree(jsonText: string): {
  success: boolean;
  root?: TreeNode;
  error?: string;
} {
  try {
    const rawData = JSON.parse(jsonText);
    const root = convertToNode(rawData, 'root', []);
    return { success: true, root };
  } catch (error) {
    return { 
      success: false, 
      error: error instanceof Error ? error.message : 'Invalid JSON' 
    };
  }
}

/**
 * 递归转换函数（核心算法）
 */
function convertToNode(
  value: any, 
  key: string, 
  path: string[]
): TreeNode {
  const id = generateUUID();
  const currentPath = [...path, key];
  
  // 判断数据类型
  if (value === null) {
    return {
      id,
      type: NodeType.NULL,
      key,
      path: currentPath,
      value: null
    };
  }
  
  if (Array.isArray(value)) {
    return {
      id,
      type: NodeType.ARRAY,
      key,
      path: currentPath,
      children: value.map((item, index) => 
        convertToNode(item, `[${index}]`, currentPath)
      )
    };
  }
  
  if (typeof value === 'object') {
    return {
      id,
      type: NodeType.OBJECT,
      key,
      path: currentPath,
      children: Object.entries(value).map(([k, v]) => 
        convertToNode(v, k, currentPath)
      )
    };
  }
  
  // 基本类型
  const type = typeof value === 'string' ? NodeType.STRING :
               typeof value === 'number' ? NodeType.NUMBER :
               NodeType.BOOLEAN;
  
  return {
    id,
    type,
    key,
    path: currentPath,
    value
  };
}
```

#### 3.2.2 反向转换函数

```typescript
/**
 * 将数据模型树转换回标准 JSON
 */
function treeToJson(node: TreeNode): any {
  if (node.type === NodeType.OBJECT) {
    const obj: Record<string, any> = {};
    (node as ContainerNode).children.forEach(child => {
      obj[child.key] = treeToJson(child);
    });
    return obj;
  }
  
  if (node.type === NodeType.ARRAY) {
    return (node as ContainerNode).children.map(child => treeToJson(child));
  }
  
  return (node as PrimitiveNode).value;
}
```

### 3.3 思维导图渲染模块

#### 3.3.1 AntV X6 图数据转换

```typescript
/**
 * 将内部数据模型转换为 X6 图数据格式
 */
function treeToGraphData(node: TreeNode): {
  nodes: X6Node[];
  edges: X6Edge[];
} {
  const nodes: X6Node[] = [];
  const edges: X6Edge[] = [];
  
  function traverse(current: TreeNode, parentId?: string) {
    // 创建节点
    const graphNode = createGraphNode(current);
    nodes.push(graphNode);
    
    // 创建与父节点的连线
    if (parentId) {
      edges.push({
        id: `edge-${parentId}-${current.id}`,
        source: parentId,
        target: current.id,
        attrs: {
          line: {
            stroke: '#C2C8D5',
            strokeWidth: 2
          }
        }
      });
    }
    
    // 递归处理子节点
    if ('children' in current) {
      current.children.forEach(child => traverse(child, current.id));
    }
  }
  
  traverse(node);
  return { nodes, edges };
}
```

### 3.4 交互与编辑模块

#### 3.4.1 编辑操作抽象层

```typescript
/**
 * 编辑操作接口（命令模式）
 */
interface EditCommand {
  execute(): void;
  undo(): void;
}

/**
 * 修改节点值
 */
class UpdateValueCommand implements EditCommand {
  constructor(
    private nodeId: string,
    private oldValue: any,
    private newValue: any,
    private newType?: NodeType
  ) {}
  
  execute() {
    const node = findNodeById(this.nodeId);
    if (isPrimitiveNode(node)) {
      node.value = this.newValue;
      if (this.newType) {
        node.type = this.newType;
      }
      syncToJson(); // 同步到 JSON 文本
      rerenderGraph(); // 重新渲染图
    }
  }
  
  undo() {
    const node = findNodeById(this.nodeId);
    if (isPrimitiveNode(node)) {
      node.value = this.oldValue;
      syncToJson();
      rerenderGraph();
    }
  }
}

/**
 * 添加子节点
 */
class AddChildCommand implements EditCommand {
  private newNodeId: string;
  
  constructor(
    private parentId: string,
    private key: string,
    private value: any,
    private type: NodeType
  ) {
    this.newNodeId = generateUUID();
  }
  
  execute() {
    const parent = findNodeById(this.parentId);
    if (isContainerNode(parent)) {
      const newNode = createNode(this.key, this.value, this.type);
      parent.children.push(newNode);
      syncToJson();
      rerenderGraph();
    }
  }
  
  undo() {
    const parent = findNodeById(this.parentId);
    if (isContainerNode(parent)) {
      parent.children = parent.children.filter(n => n.id !== this.newNodeId);
      syncToJson();
      rerenderGraph();
    }
  }
}
```

### 3.5 双向同步机制

```typescript
/**
 * 状态管理器（使用 Zustand）
 */
const useAppStore = create<AppState & Actions>((set, get) => ({
  // 状态
  jsonText: '',
  isValid: true,
  rootNode: null,
  selectedNodeId: null,
  errorMessage: null,
  
  // 操作：从 JSON 文本更新树
  updateFromJson: (text: string) => {
    const result = parseJsonToTree(text);
    if (result.success) {
      set({
        jsonText: text,
        rootNode: result.root,
        isValid: true,
        errorMessage: null
      });
    } else {
      set({
        isValid: false,
        errorMessage: result.error
      });
    }
  },
  
  // 操作：从树更新 JSON 文本
  syncToJson: () => {
    const { rootNode } = get();
    if (rootNode) {
      const json = treeToJson(rootNode);
      const text = JSON.stringify(json, null, 2);
      set({ jsonText: text });
    }
  },
  
  // 操作：执行编辑命令
  executeCommand: (command: EditCommand) => {
    command.execute();
    // 可选：维护历史栈以实现撤销/重做
  }
```

## 4. 性能优化策略

### 4.1 大数据量处理方案

#### 4.1.1 懒加载（Lazy Loading）

```typescript
/**
 * 默认仅展开前 3 层（满足 PRD 4.1 要求）
 */
function treeToGraphData(
  node: TreeNode, 
  maxDepth: number = 3
): GraphData {
  const nodes: X6Node[] = [];
  const edges: X6Edge[] = [];
  
  function traverse(current: TreeNode, depth: number, parentId?: string) {
    const graphNode = createGraphNode(current);
    
    // 超过最大深度时折叠子节点
    if (depth >= maxDepth && isContainerNode(current)) {
      graphNode.collapsed = true;
      graphNode.data.childCount = current.children.length;
    }
    
    nodes.push(graphNode);
    
    if (parentId) {
      edges.push(createEdge(parentId, current.id));
    }
    
    // 未折叠时才递归处理子节点
    if (!graphNode.collapsed && isContainerNode(current)) {
      current.children.forEach(child => 
        traverse(child, depth + 1, current.id)
      );
    }
  }
  
  traverse(node, 0);
  return { nodes, edges };
}
```

```typescript
/**
 * 根据输入内容自动推断类型
 */
function inferType(input: string): { value: any; type: NodeType } {
  // 尝试解析为数字
  const num = Number(input);
  if (!isNaN(num) && input.trim() !== '') {
    return { value: num, type: NodeType.NUMBER };
  }
  
  // 检查布尔值
  if (input === 'true') {
    return { value: true, type: NodeType.BOOLEAN };
  }
  if (input === 'false') {
    return { value: false, type: NodeType.BOOLEAN };
  }
  
  // 检查 null
  if (input === 'null') {
    return { value: null, type: NodeType.NULL };
  }
  
  // 默认为字符串
  return { value: input, type: NodeType.STRING };
}
```

```typescript
  const link = document.createElement('a');
  link.href = url;
  link.download = `mindmap_${Date.now()}.svg`;
  link.click();
  
  URL.revokeObjectURL(url);
}
```

## 9. 关键技术难点与解决方案

### 9.1 难点一：类型一致性保证（PRD 4.3）

**问题：** 确保编辑过程中不会意外改变数据类型

**解决方案：**
1. 在数据模型中强制使用 TypeScript 严格模式
2. 编辑时提供类型选择器，用户明确选择类型
3. 保存时进行类型校验，阻止不合法的类型转换
4. 使用单元测试覆盖所有类型转换场景

### 9.2 难点二：组合式分裂节点的实现（R-NODE-3）

**问题：** 如何在一个节点内实现左右分栏且保持视觉统一

**解决方案：**
使用 X6 的自定义 React 节点能力：

```typescript
Graph.registerReactComponent('split-node', (props) => {
  return (
    <div className="split-node-container">
      <div className="left-key">{props.node.key}</div>
      <div className="right-value" style={{ backgroundColor: getColor(props.node.type) }}>
        {props.node.value}
      </div>
    </div>
  );
});
```

配合 CSS Grid 或 Flexbox 实现精确分栏。

### 9.3 难点三：对称布局算法（R-LAYOUT-2）

**问题：** Dagre 默认布局可能不够对称

**解决方案：**
1. 使用 Dagre 的 `ranker` 配置调整层级分配
2. 自定义排序函数确保子节点均匀分布：

```typescript
function sortChildrenForBalance(children: TreeNode[]): TreeNode[] {
  // 按子树大小排序，大的在中间，小的在两端
  const sorted = children.sort((a, b) => {
    const sizeA = getTreeSize(a);
    const sizeB = getTreeSize(b);
    return sizeB - sizeA;
  });
  
  // 重新排列：[2, 0, 1, 3] -> [0, 1, 2, 3] 变为中心对称
  return reorderForSymmetry(sorted);
}
```

### 9.4 难点四：实时双向同步（F-OUT-1）

**问题：** 编辑器和导图的同步可能产生循环更新

**解决方案：**
使用标志位阻断循环：

```typescript
let isUpdatingFromEditor = false;
let isUpdatingFromGraph = false;

function onEditorChange(text: string) {
  if (isUpdatingFromGraph) return;
  isUpdatingFromEditor = true;
  updateFromJson(text);
  isUpdatingFromEditor = false;
}

function onGraphChange() {
  if (isUpdatingFromEditor) return;
  isUpdatingFromGraph = true;
  syncToJson();
  isUpdatingFromGraph = false;
}
```
